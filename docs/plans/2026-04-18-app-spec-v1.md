**Goal:** Build App-Spec v1 — a cross-implementation operational
specification + live-VM scenario runner. This plan executes inside the
new top-level `AppSpec/` repo; remaining tasks target paths under
`AppSpec/` and `APIAnyware-MacOS/knowledge/apps/modaliser/`.

**Architecture:** Three concentric pieces.
1. **Three conformance contracts** documented in `APIAnyware-MacOS/knowledge/apps/modaliser/`: structured
   log format (already implemented via `lib/events.rkt`), externally
   observable state (MRU / lock / XDG paths), and a shared test config
   (`AppSpec/config/test-config.scm`) every impl loads unchanged.
2. **Two Racket `#lang` dialects.** Scenario files are `#lang
   app-spec` programs; impl configs are `#lang
   app-spec/impl` modules. Registered as a collection under
   `AppSpec/app-spec/`; runner pushes `spec/` onto
   `current-library-collection-paths` at startup so no global `raco pkg
   install` is needed.
3. **Runner + SDK.** `AppSpec/runner/` drives per-scenario setup (quit old
   impl, truncate log/MRU, regrant accessibility), executes scenarios
   against a shared VM, captures artifacts on failure. `AppSpec/testanyware-sdk/`
   wraps the `testanyware` CLI + agent HTTP so the runner stays VM-agnostic.

Plus a small prelude: close three remaining log-event gaps in
Modaliser-Racket itself (`lifecycle shutdown reason=signal|error`,
implicit `[window] focus` in `activate-app!`) so the contract is fully
satisfiable by the impl under test.

**Tech Stack:** Racket CS 8+; `syntax/module-reader` for the #lang
dialects; `racket/cmdline` for runner CLI; `net/url` + `racket/port` for
agent HTTP; subprocess + place-channel for tailing events.log;
`testanyware` CLI + agent HTTP on port 8648; macOS via tart VM.

**Open questions resolved by this plan:**

1. **MRU file format** → **keep Racket `write`/`read` for v1.** No second
   impl exists; a speculative migration risks user data without benefit.
   Document the format precisely in `observable-state.md` and flag JSON
   lift as a future migration when Swift impl actually pulls on the
   seam.
2. **Config path resolution** → **env var `MODALISER_CONFIG` only.**
   Already implemented; impl configs declare `#:config-env
   "MODALISER_CONFIG"`. No CLI flag for v1.
3. **Lock file** → **document what Modaliser-Racket writes today**; do
   not standardise format yet (path + PID contents is sufficient for
   scenario assertions).
4. **Agent `/health` semantics** → **use `[lifecycle] startup` log line
   as the impl-readiness probe.** No separate Modaliser-specific health
   endpoint. `testanyware`'s `/health` remains the VM-reachability probe.

## Task 1: Emit `[lifecycle] shutdown reason=signal`

**Files:**
- Modify: `main.rkt` (signal handler install; call `close-events!` on exit)
- Test: `tests/test-lifecycle-events.rkt` (new)

Design note: Racket CS on macOS catches SIGTERM/SIGINT via
`ffi/unsafe/signal-handler` — or more portably, via `exit-handler` for
graceful-quit paths (NSApp termination, user Cmd-Q). For `kill` / `^C`,
install a handler that logs, flushes, and exits. Use `signal/signal`
from `racket/signal` (CS only) OR fall back to
`foreign-function-pointer`-based `signal(3)`.

Simplest portable: hook `(exit-handler)` chain (covers NSApp's
`[NSApp terminate:]` which ends up calling `exit`). For raw SIGTERM we
install a C `signal(2)` handler that sets a flag polled from the main
thread — but the Cocoa loop doesn't poll. Pragmatic v1: use
`signal(SIGTERM, h)` where `h` is a Racket `_cprocedure` that calls
`log-event` + `close-events!` + `_exit(0)`. Match the "FFI callback at
module top-level" pattern.

```racket
;; ── Shutdown-signal wiring ────────────────────────────────────────
;; Module-top-level so the _cprocedure survives GC (see memory.md:
;; "FFI pointers and GC: hold at module scope").

(require ffi/unsafe
         (only-in racket/os getpid))

(define signal-action
  (_cprocedure #:atomic? #t (list _int) _void))

(define _signal (get-ffi-obj "signal" #f
                             (_fun _int signal-action -> signal-action)))
(define _exit (get-ffi-obj "_exit" #f (_fun _int -> _void)))

(define SIGINT 2)
(define SIGTERM 15)

(define signal-handler-proc
  (lambda (signum)
    ;; No allocation-heavy work in the signal handler: flush + write
    ;; the shutdown line, then _exit so atexit handlers don't race.
    (log-event 'lifecycle 'shutdown 'reason 'signal)
    (close-events!)
    (_exit 128)))

(define signal-handler-fp
  (cast signal-handler-proc signal-action _fpointer))

(define (install-signal-handlers!)
  (_signal SIGTERM signal-handler-fp)
  (_signal SIGINT  signal-handler-fp))

;; ── Test hook: block on semaphore instead of NSApp-run ────────────
(define test-block-mode (getenv "MODALISER_TEST_BLOCK"))

(define (maybe-block-for-test)
  (when (equal? test-block-mode "signal")
    (install-signal-handlers!)
    (log-event 'lifecycle 'startup)
    (semaphore-wait (make-semaphore 0))))  ; blocks forever

;; Call maybe-block-for-test near the top of main, right after
;; events-init!; return without entering NSApp if it doesn't return.
```

In the main startup flow, add:
```racket

(maybe-block-for-test)           ; no-op in production
(install-signal-handlers!)       ; real run: also install for production shutdown
;; ... existing NSApp setup continues ...
```

Extend `maybe-block-for-test` and the top-level:

```racket
(define (maybe-block-for-test)
  (cond
    [(equal? test-block-mode "signal")
     (install-signal-handlers!)
     (log-event 'lifecycle 'startup)
     (semaphore-wait (make-semaphore 0))]
    [(equal? test-block-mode "raise")
     (log-event 'lifecycle 'startup)
     (error 'modaliser "synthetic test failure")]))

;; Install once at program start, around the call to maybe-block-for-test
;; and NSApp startup:
(uncaught-exception-handler
  (lambda (exn)
    (log-event 'lifecycle 'shutdown 'reason 'error
               'message (if (exn? exn) (exn-message exn) (format "~v" exn)))
    (close-events!)
    (exit 1)))
```

Place the `(uncaught-exception-handler ...)` call *before* any code that
could raise (e.g., before `events-init!` — using a temporary handler that
logs to stderr only — and then re-set with the log-aware handler after
`events-init!` succeeds). The handler must be idempotent: a crash
*inside* the handler itself must still exit.

# Modaliser Logging Contract

Every Modaliser implementation MUST emit a structured event log
consumed by App-Spec scenarios.

## File Location

- **Default path:** `$XDG_CACHE_HOME/modaliser/events.log`
  (fallback `~/.cache/modaliser/events.log`).
- **Override:** env var `MODALISER_EVENTS_LOG` takes precedence.
- **Lifecycle:** truncated on impl startup (single-writer; lock file
  guarantees no second process appends concurrently). Parent directory
  created if missing.
- **Buffering:** line-buffered; flushed after every record.

## Line Format

```
[<module>] <event-name> <key1>=<value1> <key2>=<value2>\n
```

- `<module>` and `<event-name>` are bare lowercase identifiers
  (`[a-z][a-z0-9_-]*`).
- Key-value pairs, space-separated; order is stable per event (listed
  below).
- Unknown keys MAY be added at the end of a line without breaking
  scenarios — scenarios match with anchored prefixes.

### Value encoding

| Type | Encoding | Example |
|------|----------|---------|
| boolean | bare `true` / `false` | `enabled=true` |
| exact integer | bare decimal | `pid=1234` |
| rational | decimal (may include `.`) | `delay=0.35` |
| safe symbol | bare (pattern `[A-Za-z0-9_.:/+*?<>=!@#$%^&-]+`) | `reason=menu` |
| string | double-quoted, escaped (`\\`, `\"`, `\n`, `\r`, `\t`) | `title="Untitled — TextEdit"` |
| path | string-encoded | `path="/Users/x/.config/modaliser/config.scm"` |

## Required Events (v1)

Stability: events below are **stable**; new events may be introduced
additively. Renaming or removing an event is a breaking change.

### `[lifecycle]`

| Event | Keys | When |
|-------|------|------|
| `startup` | (none) | After `events-init!` succeeds; before any other events |
| `shutdown` | `reason` = `menu` \| `signal` \| `error`; if `error`, also `message="..."` | Any terminal path |

### `[config]`

| Event | Keys | When |
|-------|------|------|
| `load` | `path="..."` | Just before evaluating config.scm |
| `loaded` | (none) | Config evaluation returned successfully |
| `missing` | `path="..."` | Config file not found |
| `error` | `message="..."` | Config evaluation raised |

### `[modal]`

| Event | Keys | When |
|-------|------|------|
| `enter` | `tree=<name>` | Leader key pressed; modal entered |
| `group` | `key=<key>` | Group-node key selected in modal |
| `exit` | `reason` = `user` \| `watchdog` \| `focus-loss` | Modal dismissed |

### `[chooser]`

| Event | Keys | When |
|-------|------|------|
| `open` | `selector="<label>"` | Chooser panel shown |
| `push` | `query="<text>"` `results=<n>` | Query entered; results recomputed |
| `close` | `reason` = `select` \| `cancel` \| `secondary-action` | Chooser dismissed |

### `[launch]`

| Event | Keys | When |
|-------|------|------|
| `bundle` | `bundle=<bundle-id>` | App launched by bundle id |
| `url` | `url="<url>"` | Non-app URL opened |

### `[window]`

| Event | Keys | When |
|-------|------|------|
| `focus` | `pid=<n>` `title="<text>"` | Window focused (explicit or implicit via `activate-app!`) |
| `move` | `x=<n>` `y=<n>` `w=<n>` `h=<n>` | Window position/size changed via Modaliser |

### `[mru]`

| Event | Keys | When |
|-------|------|------|
| `record` | `key=<remember-key>` `id="<id>"` | Item pushed to MRU front |

## Emitter Implementation Notes (non-contractual)

Modaliser-Racket implements the contract via `lib/events.rkt`. Other
implementations MAY use any mechanism; only on-disk output is observable
by scenarios.

- Emission is a no-op before `events-init!` is called — prevents unit
  tests from polluting the log.
- Filesystem errors post-init are swallowed silently (don't crash the
  app just because disk is full).
- Argument-shape errors (odd number of key/value args) ARE raised — they
  indicate a programming bug.
```

# Modaliser Observable State

Files and paths every Modaliser implementation exposes, consumable by
App-Spec scenarios via direct filesystem reads (`testanyware exec`
+ `cat`/`stat`).

## Paths (XDG-conformant on macOS)

| Path | Purpose | Format | Writer |
|------|---------|--------|--------|
| `~/.config/modaliser/config.scm` | User config; the LispKit-style flat-namespace DSL | s-expression | User (Modaliser reads only) |
| `~/.config/modaliser/mru.dat` | Per-selector MRU id lists | Racket `write`/`read` of an immutable hash (string → list) | Modaliser (on every `mru-record!`) |
| `~/.cache/modaliser/events.log` | Structured event log | See logging-contract.md | Modaliser (line-buffered, flush-per-line) |
| `~/.cache/modaliser/modaliser.pid` | Single-instance lock file | Plain text: pid as decimal, followed by newline | Modaliser (on startup; removed on clean shutdown) |

All paths honour `$XDG_CONFIG_HOME` / `$XDG_CACHE_HOME` when set.

## MRU File Format (v1)

v1 format is Racket s-expression: the file contains exactly one datum,
a `write`-serialised immutable hash. Keys are strings (the selector's
`remember` property); values are lists of id values (most recent first,
cap 50 entries).

Example (`(read)` on the file yields):

```racket
#hash(("apps" . ("com.apple.Safari" "com.apple.TextEdit" "com.apple.finder"))
      ("windows" . ("12345:Untitled" "23456:README.md")))
```

**Language-neutral lift deferred** until a second implementation exists.
Rationale: Modaliser-Racket is the only writer today; rewriting user
MRU files for a speculative future reader is premature. When Swift (or
other) impl lands, migrate to JSON (`{ "apps": ["com.apple.Safari", …]
}`) with a one-shot on-startup converter that recognises both formats
and rewrites s-expr → JSON.

## Lock File Format (v1)

Plain text: `<pid>\n`. Contents are sufficient for the single-instance
guard; scenarios use `read-file` + parse to verify only-one-instance
invariants.

Cleanup: removed on clean shutdown (menu Quit, signal); left behind on
error-exit (next startup stales-detects it by checking whether the PID
is still alive).

## Scenario Access Patterns

Scenarios read observable state through harness verbs:

- `(read-mru)` → parses `~/.config/modaliser/mru.dat` into a hash.
- `(read-file path)` → raw bytes; caller parses.
- `(expect-file path #:exists? #t)` / `(expect-file path #:absent? #t)`
  — existence assertion.
- Log assertions go through the log-specific harness verbs
  (`expect-log`, `wait-for-log`, `expect-not-log`), not `read-file`.
```

;; app-spec/impl/main.rkt — DSL surface for impl config files.

(require (for-syntax racket/base syntax/parse))

(provide impl
         (struct-out impl-spec))

(struct impl-spec (name binary config-env log-env bundle-id launch-via)
  #:prefab)

(define valid-launch-methods '(open launchctl direct))

(define-syntax (impl stx)
  (syntax-parse stx
    [(_ (~or (~once (~seq #:name       name:expr))
             (~once (~seq #:binary     binary:expr))
             (~once (~seq #:config-env config-env:expr))
             (~once (~seq #:log-env    log-env:expr))
             (~once (~seq #:bundle-id  bundle-id:expr))
             (~once (~seq #:launch-via launch-via:expr)))
        ...)

         (provide impl-spec)
         (define impl-spec
           (let ([lv launch-via])
             (unless (memq lv valid-launch-methods)
               (error 'impl "launch-via must be 'open, 'launchctl, or 'direct; got ~v" lv))
             (impl-spec name binary config-env log-env bundle-id lv))))]))

;; app-spec/main.rkt — scenario-DSL surface.

(require (for-syntax racket/base syntax/parse)
         racket/stxparam)

(provide (all-from-out racket/base)

         ;; Scenario registration
         scenario
         (struct-out scenario)
         all-scenarios
         current-scenario-registry

         ;; Input verbs (forwarded to current-driver by runner's harness)
         press type chord click-at move-mouse

         ;; Observation verbs
         expect-log wait-for-log expect-not-log
         expect-ocr expect-ax expect-no-ax expect-running-app expect-file

         ;; State verbs
         read-mru read-file kill-impl! restart-impl!

         ;; Time / sync
         wait-for-ocr wait)

;; ── Registry ─────────────────────────────────────────────────────
;; A module-level list mutated by `scenario`. Racket evaluates a
;; scenario file body top-to-bottom; `dynamic-require ... 'all-scenarios`
;; from the runner reads the final list after evaluation completes.

(struct scenario (name description thunk) #:prefab)

(define all-scenarios '())

;; current-scenario-registry lets tests override destination.
(define current-scenario-registry
  (make-parameter
   (lambda (s) (set! all-scenarios (append all-scenarios (list s))))))

(define-syntax (scenario stx)
  (syntax-parse stx
    [(_ name:expr (~optional (~seq #:description desc:expr)
                             #:defaults ([desc #'""]))
        body:expr ...+)
     #'((current-scenario-registry)
        (scenario name desc (lambda () body ...)))]))

#lang racket/base
;; testanyware-sdk/exec.rkt — thin wrapper over `testanyware exec`.

(require racket/port
         racket/system
         racket/string)

(provide gv-exec
         current-testanyware-runner)

;; Injection point for tests: a function (list-of-string -> rc out err).
(define current-testanyware-runner
  (make-parameter default-testanyware-runner))

(define (default-testanyware-runner args)
  (define-values (sp out-p err-p in-p)
    (apply subprocess #f #f #f "/usr/local/bin/testanyware" args))
  (close-output-port in-p)
  (define out (port->string out-p))
  (define err (port->string err-p))
  (close-input-port out-p)
  (close-input-port err-p)
  (subprocess-wait sp)
  (values (subprocess-status sp) out err))

(define (gv-exec cmd #:vm [vm (getenv "TESTANYWARE_VM_ID")])
  (unless vm
    (error 'gv-exec "no VM id: pass #:vm or set TESTANYWARE_VM_ID"))
  ((current-testanyware-runner) (list "exec" "--vm" vm cmd)))

#lang racket/base
(require "exec.rkt")

(provide quit-impl!
         reset-tcc!
         grant-accessibility!
         wait-ready)

(define (quit-impl! bundle-id #:vm [vm #f])
  ;; Prefer `osascript -e 'tell app id "..." to quit'` over pkill —
  ;; graceful shutdown lets the impl emit `[lifecycle] shutdown
  ;; reason=menu` (via applicationWillTerminate) before exit.
  (gv-exec
    (format "osascript -e 'tell application id \"~a\" to quit' 2>/dev/null || true"
            bundle-id)
    #:vm vm))

(define (reset-tcc! bundle-id #:vm [vm #f])
  ;; Clears Accessibility, ScreenCapture, etc. for one bundle id so
  ;; scenarios start from a clean TCC state.
  (gv-exec (format "tccutil reset All ~a 2>/dev/null || true" bundle-id)
           #:vm vm))

(define (grant-accessibility! bundle-id #:vm [vm #f])
  ;; Insert an Accessibility grant into TCC.db. Admin DB write; VM-only.
  (gv-exec
    (format
     (string-append
      "sudo sqlite3 /Library/Application\\ Support/com.apple.TCC/TCC.db "
      "\"INSERT OR REPLACE INTO access "
      "(service, client, client_type, auth_value, auth_reason, "
      "auth_version, indirect_object_identifier, flags, last_modified) "
      "VALUES ('kTCCServiceAccessibility', '~a', 0, 2, 3, 1, 'UNUSED', 0, ~a);\""
      " 2>/dev/null || true")
     bundle-id
     (current-seconds))
    #:vm vm))

(define (wait-ready bundle-id
                    #:vm          [vm #f]
                    #:events-path [events-path
                                    "/Users/admin/.cache/modaliser/events.log"]
                    #:timeout     [timeout-s 10])
  ;; Poll the VM's events.log for `[lifecycle] startup` within timeout.
  ;; This is the Modaliser-specific readiness probe (design Q4).
  (define deadline (+ (current-inexact-milliseconds) (* 1000 timeout-s)))
  (let loop ()
    (define-values (rc out err)
      (gv-exec (format "cat ~a 2>/dev/null || true" events-path) #:vm vm))
    (cond
      [(regexp-match? #px"\\[lifecycle\\] startup" out) #t]
      [(> (current-inexact-milliseconds) deadline)
       (error 'wait-ready "impl ~a did not emit [lifecycle] startup within ~as"
              bundle-id timeout-s)]
      [else (sleep 0.2) (loop)])))

  (cond
    [(not (hash? node)) #f]
    [(and (equal? (hash-ref node 'AXRole #f)
                  (symbol->string role))
          (or (not title)
              (equal? (hash-ref node 'AXTitle #f) title)))
